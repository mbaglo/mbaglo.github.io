Building a Pulse Timing Measurement Tool with STM32 Input Capture

2026-05-29

Accurate timing is one of the most important parts of embedded systems design. In this project, I built a pulse timing measurement tool using an STM32 microcontroller to detect an incoming signal, measure its timing, and analyze its stability. The system leverages TIM2 input capture to record pulse events with microsecond precision.

Project Goal

  • Detect whether an input signal is present
  • Measure pulse timing with microsecond resolution
  • Allow the user to define a timing range through UART
  • Collect 1000 valid measurements
  • Summarize the results in a histogram-style table

This makes the project useful for evaluating how stable or noisy a repeated signal is over time.

How It Works

The firmware begins with a power-on self-test (POST). During this step, input capture is started and the system checks for signal activity. If a capture event occurs, the system reports success and moves into the measurement phase. Otherwise, it notifies the user via UART.

After POST, the program displays the current lower and upper pulse-width limits through UART2. By default, these are 950 µs and 1050 µs, but the user can adjust them via serial commands.

Pulse periods are measured by recording two consecutive rising edge timestamps on TIM2. The difference between these values gives the pulse period in microseconds.

Timing and Signal Processing

A timer prescaler of 79 reduces the clock so the counter increments once every microsecond — simplifying period calculation. If a captured pulse width falls within the defined window, it is categorized into one of 101 histogram buckets representing each value between the lower and upper bounds.

This approach visualizes the complete distribution of pulse timings, making it easy to spot timing variation, clustering, or outliers.

Output

After 1000 valid measurements, input capture stops and the histogram is sent over UART, listing each microsecond value and how often it occurred.

What I Learned

  • Configuring STM32 timers for precise measurement
  • Using input capture interrupts for event timing
  • Building a UART-based user interface
  • Organizing measurement data into histogram buckets
  • Combining hardware peripherals to create practical diagnostic tools

This project demonstrated how low-level STM32 peripherals can be combined to build a complete measurement application with flexible data collection and user interaction.

Possible Improvements

  • Explicit handling for timer counter overflow
  • Input filtering for noisy or unstable signals
  • Configurable upper/lower bounds
  • CSV-style serial output for easier logging
  • PC-based graphical visualization

Final Thoughts

This project is a strong example of how embedded systems can be used for precise real-time measurement. By combining STM32 timers, interrupt-based capture, and UART communication, you can move beyond basic microcontroller use and start building tools that solve meaningful measurement and debugging problems.
